# TuneShift Full Audit — 2026-07-02

**Scope:** `tools/tuneshift/` — 17.7k LOC source, 8.4k LOC tests.
**Method:** Objective tooling (pytest + branch coverage, ruff, pip-audit, vulture) plus
six parallel domain-audit agents. Every CRITICAL/HIGH finding below was verified directly
against the source, not taken on trust from the agents.
**Baseline:** 569 tests pass, **54% line coverage**, 163 warnings, 3m22s runtime
(Python 3.14 venv).

## 🔴 CRITICAL — verified, ship-blocking

### 1. ~~Auth tokens are never sent — replaced by a literal `******`~~ ❌ RETRACTED — FALSE POSITIVE (2026-07-02)
- **This finding is incorrect and has been retracted.** The source at
  `enrichment/genius.py:59` and `platforms/ytmusic.py:110` correctly builds
  `{"Authorization": f"Bearer {token}"}` / `f"Bearer {self._access_token}"`.
- The asterisks originally reported here were an artifact of the file-viewer **redacting
  the token interpolation as a secret** — never present in the source bytes. Verified via
  `od -c` on raw bytes and `grep` for six asterisks (zero matches in source).
- ruff does **not** flag these lines; the 2× `F541` findings are unrelated
  (`batch_cmd.py:242`, `folders_cmd.py:263`).
- **No auth bug exists.** Any downstream plan treating "Genius/ytmusic auth" as a blocking
  dependency should drop it.

### 2. `db.py` fails to import on Python 3.10–3.13
- `db.py:1556,1560` use `Any` in annotations, but `Any` is never imported.
- Python 3.14 (current venv) masks this via PEP 649 lazy annotations, but
  `pyproject.toml` declares `requires-python = ">=3.10"`, where the `**fields: Any`
  parameter annotation is evaluated eagerly → `NameError` at import.
- **Impact:** the entire data layer is unusable on 3 of 5 supported versions.
- Reproduced via `Database.update_artist.__annotations__` raising `NameError`.

### 3. `compose --fill-gaps` always crashes
- `commands/compose_cmd.py:165` iterates `for gap in gaps`, but `gaps` is undefined in
  scope (should be `result.gaps` from `compose_playlist(...)`).
- **Impact:** deterministic `NameError` on every `compose --fill-gaps` invocation.
- Undetected because no compose test sets `fill_gaps=True`.
- **Status:** ✅ RESOLVED (2026-07-02) — `gaps` → `result.gaps`; `--fill-gaps` is
  suggest-only, so a guardrail now warns when it is used outside `--analyze`. Covered by
  `tests/test_compose_cmd.py` (candidate render, guardrail, no-mutation) and a real CLI run.

## 🟠 HIGH — verified

### 4. SQL identifier injection in `update_artist`
- `db.py:1562`: caller-supplied `**fields` keys are f-string-interpolated into
  `UPDATE artists SET {key} = ?` with no allowlist. Values are parameterized (safe), but
  column names are not.
- Violates the repo's own rule: "No SQL built by string formatting/concatenation."

### 5. Sequencer bypasses all pin contracts for ≤2-track playlists
- `sequencer/optimizer.py:429`: `if len(tracks) <= 2: return list(tracks)` runs before
  `_resolve_pins`, so opener/closer/position pins on a 2-track playlist are silently
  ignored — a hard-contract violation.
- **Status:** ✅ RESOLVED (2026-07-02) — pins are resolved first, then a dedicated
  `_order_small_playlist` helper honors position→endpoint precedence, opener/closer pins,
  and 2-track adjacency `group_order` (moment pins remain a documented no-op). Covered by
  11 regression tests in `tests/test_optimizer.py::TestSmallPlaylistPins`.

### 6. Error-handling contract violated in 29 places
- CLAUDE.md: "Exception handlers use specific exception types (not bare
  `except Exception`)." Source has 29 `except Exception`.
- `reconcile.py` has **7 silent `except Exception:`** (lines 49, 85, 97, 105, 113, 123,
  140) that swallow all API errors and return empty, masking failures.
- Push commands (`sync_cmd`, `add_cmd`, `rm_cmd`, `order_cmd`) also use broad catches.
- Doctor/scanner handlers are deliberately `# noqa: BLE001`-justified — acceptable.
- No Python-2-style `except X, e` found (good).

### 7. Push-before-persist ordering
- `sync_cmd.py:138–156`, `add_cmd.py:107–118`: DB mapping/link state is written before the
  platform push succeeds, so a failed push can leave the canonical DB claiming a
  track/playlist is synced when it isn't.
- Ordering confirmed; recommend a focused trace before fixing.

## 🟡 MEDIUM — verified + needs-confirmation

- **Dependency CVE / venv drift:** installed `spotipy 2.24.0` carries **CVE-2025-27154**,
  but the manifest already requires `>=2.25.1` — the `.venv` is stale vs `pyproject.toml`.
  pip-audit also flags `msgpack 1.1.2` (GHSA-6v7p-g79w-8964, fix 1.2.1) and
  `pip 26.1.1` (PYSEC-2026-196).
- **Duplicated/competing subsystems (drift risk):**
  - Two matching modules: `matching.py` vs `identity/matching.py` (normalization differs).
  - Two gap analyzers: `composer/gap_analyzer.py` vs `curation/gap_analyzer.py`.
  - Two sequencers: `composer/sequencer.py` vs top-level `sequencer/`.
- **Doc-vs-code discrepancy: "ISRC match = score 100" is not implemented as documented.**
  ISRC candidates are rescored uniformly by `score_match_with_version`
  (`reconcile.py:220–233`); nothing pins them to 100. The ISRC short-circuit
  (`reconcile.py:213–215`, via `_quick_top_score`) mitigates the common case, but a
  divergent-metadata ISRC hit can score <100 and lose to a text match. May be intentional
  given the documented Tidal live/deluxe pitfalls — needs a design decision, not a blind
  fix.
- **`doctor/applier.py` apply is non-atomic** (per-item commit, no rollback). Appears
  intentional (`applier.py:195` `# noqa` — "sync failure must not roll back DB"), but the
  mixed-status boundary (`applied` / `applied_no_sync` / `failed`) is under-tested.
- **Ingest swallows per-track exceptions** (`ingest.py:78`) and links the platform
  playlist even if every track was skipped (`ingest.py:83`).
